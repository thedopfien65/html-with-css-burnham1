Shell Command Cheat Sheet

A quick reference for some commonly used shell commands.

Unix Commands

Command Usage Example
ls Lists the contents of a directory. Useful for seeing what you can access easily. $ ls
Documents Desktop MyFolder
tree Lists directories in a tree format. Useful for seeing how your directory is structured. $ tree
(directory tree display)
file <filename> Used for determining what kind of file something is. It deduces what kind of file it is based on what is contained inside and does not use the file extension. $ file myFile
myFile.txt: news or mail text
type <command> Used to determine command types, such as a built in command vs a program specific command. $ type ls
ls is a shell built in
apropos <command> Shows you information about a command, such as where it is stored and what it is used for $ apropos ls
(command data display)
man <command> Shows the manual entry on a command, useful for knowing usage that may be specific to your version or programs. $ man ls
(ls manual documentation)
pwd Acronym for Print Working Directory. It shows you what directory you are currently in. $ pwd
myDirectory
cd <directoryName> Acronym for change directory. It lets you move around in the directory tree. $ cd someDirectory
mkdir <directoryName> Shorthand for Make Directory. Allows you to create new directories. $ mkdir newDirectory
echo <content> Allows you to print stuff to the command line. Useful for getting information from scripts for debugging purposes. $ echo Hellew Werld
Hellew Werld
cat <fileName> Outputs the contents of a file to the command line. Usually you will only see the end of a file. $cat myFile
(myFile contents)
less Updated version of the older "more" command. Allows you to display different pages of a file, and has several options for doing so. $ less myFile
(file Contents)
